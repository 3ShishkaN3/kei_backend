# Kei Backend — Сердце платформы Keisenpai

Бэкенд Keisenpai — это приложение на Django 5.1.7, построенное по принципу **модульного монолита**. Оно объединяет в себе управление пользователями, курсами, контентом, прогрессом и сложную систему достижений с интеграцией ИИ-технологий.

## Архитектура бэкенда

Проект разработан с учетом масштабируемости и слабой связанности компонентов на основе событийно-ориентированной архитектуры.

### Модульный монолит
Каждый сервис является отдельным Django-приложением. Это позволяет:
*   Изолировать логику домена
*   Упростить тестирование
*   В будущем легко вынести любой сервис в отдельный микросервис

### Событийно-ориентированная модель (Apache Kafka)
Взаимодействие между сервисами происходит асинхронно через **Apache Kafka**:
*   **Производители**: Приложения отправляют данные о событиях (например, `lesson_completed`) в Kafka
*   **Потребители**: Специальные Django-команды прослушивают топики и обновляют прогресс, выдают достижения или уведомляют пользователя
*   **Топики**: Основные топики — `progress_events`, `achievement_events`, `notifications`

### Асинхронные задачи (Celery + Redis)
Для задач, которые не связаны напрямую с потоком событий, используется Celery:
*   Отправка Email
*   Проверка тестов с помощью LLM (Google Gemini)
*   Периодические задачи по расписанию (Celery Beat)

### WebSocket и real-time коммуникация
*   **Channels**: Django Channels для WebSocket-соединений
*   **Redis**: как channel layer для real-time коммуникаций
*   **ИИ-интеграция**: WebSocket-потребитель для голосового взаимодействия с Google Gemini Live API

## Состав сервисов (Apps)

### Основные сервисы
1.  **`auth_service`**: Управление пользователями, ролями (Админ, Учитель, Помощник, Студент), регистрация и аутентификация. Включает защиту от подбора паролей (django-axes) и подтверждение email.
2.  **`user_service`**: Профили пользователей, настройки и аватары
3.  **`course_service`**: Управление каталогом курсов и записью студентов
4.  **`lesson_service`**: Организация уроков, разделов и структуры контента
5.  **`exam_service`**: Система экзаменов с прокторингом, таймерами и секциями

### Контент и тестирование
6.  **`material_service`**: Хранилище учебных материалов и продвинутая система тестирования:
    *   **Материалы**: текст, изображения, аудио, видео, документы
    *   **Типы тестов**: MCQ (один/несколько ответов), свободный текст, порядок слов, перетаскивание, произношение, правописание, AI-разговоры, прописи иероглифов
    *   **ИИ-оценка**: автоматическая проверка свободных ответов через Google Gemini
7.  **`dict_service`**: Словарная система, карточки слов и аудио-произношения

### Прогресс и геймификация  
8.  **`progress_service`**: Многоуровневое отслеживание успеваемости (курсы, уроки, разделы, тесты)
9.  **`achievement_service`**: Система графов и правил для автоматической выдачи достижений с компилятором правил
10. **`bonus_service`**: Система поощрений и бонусов

### Инфраструктурные сервисы
11. **`notification_service`**: Центр уведомлений (WebPush, In-app) с VAPID
12. **`telegram_service`**: Интеграция с ботом для учителей и уведомлений
13. **`calendar_service`**: Календарь обучения и напоминания

## Технологический стек

### Core
*   **Python 3.11.11** с Django 5.1.7
*   **PostgreSQL** как основная БД
*   **Redis** для Celery и Channels
*   **Apache Kafka** для событийной шины

### API и коммуникации
*   **Django REST Framework** с drf-nested-routers
*   **Swagger/OpenAPI** документация
*   **Django Channels** для WebSocket
*   **WebPush** для push-уведомлений

### ИИ и ML
*   **Google Generative AI (Gemini 2.5 Flash Lite)** для оценки текстовых ответов
*   **gRPC** для интеграции с сервисом прописей иероглифов
*   **WebSocket с Gemini Live API** для голосового AI-общения

### Инфраструктура
*   **Docker** с multi-stage билдом
*   **S3** хранилище (с поддержкой локальных файлов)
*   **Sentry** для мониторинга ошибок
*   **Axes** для защиты от брутфорса

## Основные команды

### Миграции БД
```bash
python manage.py makemigrations
python manage.py migrate
```

### Запуск потребителей Kafka (каждый в своем процессе)
```bash
python manage.py consume_progress_events
python manage.py consume_achievement_events
```

### Celery
```bash
# Worker
celery -A kei_backend worker -l info

# Beat scheduler  
celery -A kei_backend beat -l info
```

### WebSocket сервер
```bash
daphne kei_backend.asgi.application -p 8000
```

## Важные компоненты

### ИИ-интеграция
*   **Оценка текстов**: Google Gemini для проверки заданий со свободным ответом
*   **Голосовой AI**: WebSocket-соединение с Gemini Live API для реального времени
*   **Прописи иероглифов**: gRPC-сервис для распознавания рукописного ввода

### Утилиты и инфраструктура
*   **Kafka**: Централизованная функция `send_to_kafka` в `kei_backend/utils.py`

*   **Health check**: Эндпоинт `/health/` для мониторинга состояния
*   **Безопасность**: CSRF-защита, CORS, rate limiting, защита от брутфорса

### Хранилище
*   **S3**: AWS S3 совместимое хранилище для production
*   **Local storage**: Локальные файлы для разработки
*   **Оптимизация изображений**: автоматическое сжатие и конвертация в JPEG

## Переменные окружения

## Особенности архитектуры

### Событийная модель
Все значимые события (завершение урока, сдача теста, получение достижения) отправляются в Kafka и обрабатываются асинхронно. Это обеспечивает:

*   **Надежность**: события не теряются при сбоях
*   **Масштабируемость**: потребители можно масштабировать независимо
*   **Расширяемость**: новые сервисы могут подписываться на существующие события

### Многоуровневый прогресс
Система отслеживает прогресс на 4 уровнях:
1. **Общий прогресс** пользователя
2. **Прогресс по курсам**  
3. **Прогресс по урокам**
4. **Прогресс по разделам и тестам**

### Гибкая система тестирования
Поддерживает 9 типов тестов с автоматической оценкой:
*   **Автоматические**: MCQ, порядок слов, перетаскивание
*   **ИИ-оценка**: свободный текст, произношение, AI-разговоры
*   **Специализированные**: прописи иероглифов, правописание

## Разработка

### Локальный запуск
```bash
# Установка зависимостей
pip install -r requirements.txt

# Миграции
python manage.py migrate

# Создание суперпользователя
python manage.py createsuperuser

# Запуск development сервера
python manage.py runserver
```

### Тестирование
```bash
# Запуск всех тестов
python manage.py test

# Тесты конкретного сервиса
python manage.py test material_service
```
